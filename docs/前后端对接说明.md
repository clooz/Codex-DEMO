# 易普工业大学官网 - 前后端对接说明

## 数据源原则（必读）

**所有会变的业务数据一律只来自接口/数据库，前端不写死。** 以后加新页面也按此约定执行。

- **前端**：只负责「如何取数」（调用后端接口）和「如何展示」（根据接口返回渲染）。不在 HTML/JS 中写死新闻、统计、院系、招生日程等业务数据；接口失败时展示「加载失败」等提示，不展示假数据。
- **后端**：业务数据接口**仅从数据库返回数据**。数据库查询失败时返回 `success: false` 及错误信息，不返回兜底假数据，以便前端明确区分「有真实数据」与「接口异常」。
- **数据库**：业务内容（新闻、首页统计、院系、重要节点等）的唯一真实来源。建表与初始数据见下文，对接前请先执行对应 seed 脚本。

这样可避免「先做前端模拟、后接后端」时出现数据来源混乱；对接完成后，首页与列表类数据均来自数据库。

## 一、功能与对接一览

| 功能 | 前端页面/位置 | 后端接口 | 说明 |
|------|----------------|----------|------|
| 用户登录 | `frontend/login.html` | `POST /login` | 提交用户名、密码，校验 users 表（bcrypt），成功写 localStorage 并跳转首页 |
| 登录状态展示 | 全站页头（main.js） | 无 | 读取 localStorage 的 auth_token、user_info，显示用户名或「登录」入口 |
| 校园开放日预约 | `frontend/opencampus.html` | `POST /api/reservations` | 提交预约表单，写入 reservations 表 |
| **预约查询** | `frontend/reservation-query.html` | `GET /api/reservations?contact_phone=手机号&date=可选` | 输入预约时留的手机号（可选日期）查询预约记录，与预约形成「增+查」闭环 |
| **在线留言** | `frontend/undergraduate.html` 招生办公室侧栏「在线留言」表单 | `POST /api/inquiries` | 姓名、电话、咨询类型、内容提交后写入 inquiries 表 |
| **查询我的留言** | `frontend/inquiry-query.html` | `GET /api/inquiries?phone=手机号` | 输入留言时留的手机号查询该手机号的留言列表及回复，与留言形成「增+查」闭环 |
| **留言管理（演示）** | `frontend/inquiry-manage.html` | `GET /api/inquiries`、`PATCH /api/inquiries/:id` | 列表全部留言，支持回复留言、标记已处理（改 Update），便于演示 |
| 首页新闻动态 | `frontend/index.html` 新闻列表 | `GET /api/news` | 无写死数据；加载时请求接口，成功则渲染，失败则显示「加载失败」 |
| 更多新闻 | `frontend/news.html` | `GET /api/news?limit=10&page=1` | 列表与分页均来自接口；点击跳转 `news-detail.html?id=xxx` |
| 新闻详情 | `frontend/news-detail.html` | `GET /api/news/:id` | 仅从接口取该条新闻（含 body 正文）；无 id 时跳回 `news.html` |
| **首页实力统计** | `frontend/index.html` 数据见证实力 | `GET /api/stats` | 数字、单位、标题、副标题均来自数据库，前端仅渲染 |
| **院系与学科** | `frontend/index.html` 院系与学科 | `GET /api/colleges` | 院系名称、简介、图片、链接均来自数据库，前端仅渲染 |
| **重要节点日程** | `frontend/index.html` 重要节点 | `GET /api/key-dates` | 招生/入学日程来自数据库，前端仅渲染 |
| 预约入口 | `frontend/index.html` 招生日程「预约参观」 | 无 | 链接到 `opencampus.html` |
| 站内搜索 | 全站搜索框（main.js） | 无 | 仅前端：历史与建议存 localStorage，暂不请求后端 |

## 二、本地运行前端（避免 172.16.0.0 超时）

用 `npx serve` 时，终端可能显示 **172.16.0.0** 等局域网地址，用该地址访问容易 **ERR_CONNECTION_TIMED_OUT**。请改用本机地址：

- **在浏览器中访问**：`http://localhost:5000` 或 `http://127.0.0.1:5000`（端口以终端显示为准，常见为 3000、5000）。
- **推荐启动方式**（只监听本机，不暴露局域网）：
  ```bash
  cd frontend
  npx serve -l tcp://localhost:5000
  ```
  然后在浏览器打开：**http://localhost:5000**

若已用 `npx serve` 启动且终端显示 172.x 地址，不要点终端里的链接，直接在浏览器地址栏输入 **http://localhost:端口号** 即可。

## 三、前端 API 配置

- 所有请求基地址由 **`frontend/js/config.js`** 统一配置：`window.API_BASE = 'http://localhost:3000'`。
- 登录页、预约页、首页新闻/统计/院系/重要节点均通过该配置发起请求；更换环境时只需改 `config.js`。

## 四、后端接口汇总

- 基础地址：`http://localhost:3000`（默认）
- **POST /login**：登录，body `{ username, password }`，返回 `{ success: true/false }`。
- **POST /api/reservations**：提交预约，必填 `student_name`、`contact_phone`、`reservation_date`，返回 `{ success: true, id }`。
- **GET /api/reservations**：预约列表，可选 `?date=YYYY-MM-DD`、`?contact_phone=手机号`（按手机号筛选）、`?limit=50`。
- **GET /api/news**：新闻列表，可选 `?limit=10&page=1`，返回 `{ success: true, data: [{ id, title, subtitle, summary, date_display, link }], total }`。失败返回 `success: false`，无兜底假数据。
- **GET /api/news/:id**：新闻详情，返回 `{ success: true, data: { id, title, subtitle, summary, body, date_display, link } }`。数据仅来自数据库。
- **GET /api/stats**：首页实力统计，返回 `{ success: true, data: [{ id, value, unit, label, description }] }`。数据仅来自数据库。
- **GET /api/colleges**：院系与学科，返回 `{ success: true, data: [{ id, name, description, image_url, link }] }`。数据仅来自数据库。
- **GET /api/key-dates**：重要节点日程，返回 `{ success: true, data: [{ id, tag, date_display, datetime, label, description, link, card_type }] }`。数据仅来自数据库。
- **POST /api/inquiries**：在线留言，必填 `name`、`phone`、`content`，可选 `inquiry_type`，返回 `{ success: true, id }`。
- **GET /api/inquiries**：查询留言，可选 `?phone=手机号`（按手机号筛选，用户查自己的）、不传则返回全部（管理用）；可选 `?limit=50`。
- **PATCH /api/inquiries/:id**：回复留言或标记已处理（改），body 可选 `admin_reply`、`status`（`replied`/`processed`）。
- **GET /health**：健康检查。

## 五、数据库表

- **users**：登录账号（username、password 为 bcrypt 哈希）。
- **reservations**：开放日预约（学生姓名、家长、电话、日期、人数、学校、年级、意向等）。
- **news**：新闻（id、title、subtitle、summary、body、date_display、link、sort_order）。正文在 body 字段。
- **homepage_stats**：首页实力数据（value、unit、label、description、sort_order）。
- **colleges**：院系与学科（name、description、image_url、link、sort_order）。
- **key_dates**：重要节点日程（tag、date_display、datetime、label、description、link、card_type、sort_order）。
- **inquiries**：招生咨询在线留言（name、phone、inquiry_type、content、status、admin_reply、replied_at、created_at）。status：pending/replied/processed。

建表与初始数据：

- 用户：`npm run seed`
- 预约表：`npm run seed:reservations`
- 新闻：`npm run seed:news`
- 首页统计：`npm run seed:stats`
- 院系：`npm run seed:colleges`
- 重要节点：`npm run seed:key-dates`
- 留言表：`npm run seed:inquiries`
- 一次性初始化首页三块：`npm run seed:homepage`（依次执行 stats、colleges、key-dates）
