# 易普工业大学官网首页 — 设计规范与线框图

**角色**：UI/UX Designer  
**目标**：现代化、交互流畅、视觉新颖  
**依据**：产品需求与功能清单（Senior Product Manager）

---

## 一、设计定位与风格关键词

| 关键词 | 说明 |
|--------|------|
| **现代** | 大留白、清晰层级、无冗余装饰、偏几何与排版驱动 |
| **流畅** | 滚动视差、缓动动画、hover/焦点反馈统一、无生硬跳变 |
| **新颖** | 非传统“红蓝校徽”官网感，用色彩与动效建立记忆点 |

**整体气质**：专业、开放、科技感，略带学术与工业的稳重，同时通过动效与排版显得年轻、易接近。

---

## 二、视觉系统（Design Tokens）

### 2.1 色彩

- **主色 Primary**  
  - 主色：`#0D47A1`（深蓝，偏科技与信任）  
  - 主色浅：`#1565C0`（悬停/链接）  
  - 主色更浅：`#E3F2FD`（浅底、标签背景）

- **强调色 Accent**  
  - 琥珀/橙：`#FF6F00` 或 `#E65100`（用于 CTA、数字高亮、重要入口）  
  - 与主蓝形成对比，避免“全蓝”的陈旧感

- **中性色 Neutral**  
  - 正文：`#1A1A2E`  
  - 次要文案：`#5C5C6D`  
  - 边框/分割：`#E8E8ED`  
  - 背景：主背景 `#FAFAFC`，区块交替可用 `#FFFFFF`

- **渐变（用于 Banner、卡片、背景装饰）**  
  - 主渐变：`linear-gradient(135deg, #0D47A1 0%, #1565C0 50%, #1976D2 100%)`  
  - 柔和装饰：`linear-gradient(180deg, transparent, rgba(13,71,161,0.03))`

### 2.2 字体

- **标题**：无衬线，偏几何。  
  - 推荐：**Noto Sans SC** 或 **PingFang SC**，字重 600–700。  
  - 备选：**Inter**、**DM Sans**（英/数）。

- **正文**：同一字体族，字重 400，行高 1.6–1.75。

- **字号建议（桌面）**  
  - Hero 主标题：36px–48px  
  - 区块标题：24px–28px  
  - 卡片标题：18px–20px  
  - 正文：16px  
  - 辅助/说明：14px  

- **移动端**：整体缩小约 2–4px，保持层级比例。

### 2.3 圆角与阴影

- **圆角**  
  - 卡片/按钮：`12px`  
  - 输入框/标签：`8px`  
  - 大区块/弹层：`16px` 或 `24px`

- **阴影**  
  - 卡片默认：`0 2px 12px rgba(0,0,0,0.06)`  
  - 悬停：`0 8px 24px rgba(0,0,0,0.08)`  
  - 导航/浮层：`0 4px 20px rgba(0,0,0,0.08)`

### 2.4 间距系统

- 基础单位：`8px`。  
- 常用：`8 / 16 / 24 / 32 / 48 / 64 / 96`。  
- 区块上下间距：`64px–96px`（桌面），移动端 `48px–64px`。  
- 内容最大宽度：`1200px`，左右留白自适应。

## 三、布局与线框图（按模块）

### 3.1 顶部导航栏

- **布局**  
  - 全宽，高度约 `64px`（桌面）。  
  - 左侧：Logo + 校名（可简写）。  
  - 中间/偏右：主导航（学校概况、机构设置、人才培养、科学研究、招生就业、国际交流、校园生活）。  
  - 右侧：搜索图标、移动端汉堡菜单。

- **交互与动效**  
  - 初始：背景透明或半透明（如 `rgba(255,255,255,0.9)` + 毛玻璃 `backdrop-filter: blur(12px)`）。  
  - 滚动超过一屏：背景变为纯白/浅灰 + 轻阴影，文字与 Logo 保持对比度。  
  - 导航项：hover 时下划线或背景高亮，过渡 `0.2s ease`。  
  - 移动端：点击汉堡展开全屏或侧滑菜单，菜单项带轻微 stagger 进入动画。

- **组件要点**  
  - 可点击区域 ≥ 44px（触屏友好）。  
  - 当前页可加细线或色块指示。

---

### 3.2 Hero Banner（主视觉）

- **布局**  
  - 全宽，高度约 `70vh`（桌面），最小高度 `480px`。  
  - 背景：大图 + 自上而下渐变遮罩（深色到透明），或纯渐变 + 抽象几何/网格装饰。  
  - 内容区：居中或偏左，最大宽度约 `640px`。  
  - 主标题 1 行 + 副标题 1–2 行 + 2 个 CTA 按钮（如「招生信息」「了解更多」）。

- **若为轮播**  
  - 3 张以内，自动轮播 5–6s，带圆点指示器与左右箭头。  
  - 切换：淡入淡出或轻微位移（如 `translateX` + `opacity`），时长约 0.5s ease-out。

- **动效**  
  - 首屏进入：标题与副标题、按钮依次淡入上移（stagger 各 0.1–0.15s）。  
  - 可选：背景缓慢位移（parallax）或渐变微动，增强“流畅”感。

---

### 3.3 快捷入口（Icon Cards）

- **布局**  
  - 在 Hero 下方，单行 6 个（桌面）；平板 3+3；手机 2 列 3 行。  
  - 每项：圆角卡片（或仅图标+文字，无边框），图标 48px–56px，标题 16–18px，可选副标题 14px。  
  - 卡片内边距：24px，间距 16–24px。

- **交互**  
  - Hover：卡片轻微上浮（`translateY(-4px)`）+ 阴影加深，图标可轻微放大或变色，过渡 0.25s ease。  
  - 点击：整卡可带 scale(0.98) 的按下反馈。

- **视觉**  
  - 图标统一风格（线性或面性二选一），主色/强调色区分类型。  
  - 背景可与页面主背景或浅色块区分，形成一块清晰“功能区”。

---

### 3.4 学校简介

- **布局**  
  - 两栏（桌面）：左图右文或右图左文，图约 40% 宽，文 60%。  
  - 移动端：图上文下。  
  - 标题 + 2–3 段正文 +「了解更多」链接/按钮。

- **动效**  
  - 进入视口时：文字与图片分别淡入+自下而上短位移（如 24px），可用 `Intersection Observer` 触发，duration 0.5–0.6s。

---

### 3.5 新闻/通知动态

- **布局**  
  - 列表式：左侧日期（小号字），右侧标题 + 摘要（可选），一行一条，分隔线或留白分隔。  
  - 右侧或下方「更多」链接。

- **交互**  
  - 列表项 hover：背景浅色或左边框高亮，过渡 0.2s。  
  - 可选：列表整体进入视口时，条目依次淡入上移（stagger）。

---

### 3.6 数据展示（数字亮点）

- **布局**  
  - 一行 3–4 个数字块，居中或均匀分布。  
  - 每块：大号数字（36–48px）+ 单位/说明（如「个学科」「项专利」）。

- **动效（新颖点）**  
  - 进入视口时：数字从 0 递增到目标值（CountUp），时长约 1.2s，ease-out。  
  - 数字块可带轻微 scale 或 opacity 进入，与 CountUp 同步。

---

### 3.7 院系/学科展示

- **布局**  
  - 卡片网格：桌面 3 列，平板 2 列，手机 1 列。  
  - 每卡：封面图或图标 + 标题 + 简短描述 +「了解」链接。  
  - 卡片统一高宽比（如 4:3 或 1:1），圆角 12px。

- **交互**  
  - Hover：图轻微缩放（如 1.05）、卡片阴影加深。  
  - 进入视口：卡片依次或成组淡入上移（stagger），营造流畅感。

---

### 3.8 招生/就业入口

- **布局**  
  - 一块 CTA 区：背景用主色或渐变，内嵌 2–3 个明显按钮（本科招生、研究生、就业）。  
  - 文案简短有力（如「加入我们」「开启未来」）。

- **交互**  
  - 按钮 hover：亮度/对比度微调或轻微放大，过渡 0.2s。

---

### 3.9 校园风采（图库）

- **布局**  
  - 网格 2x3 或 3x2，或不对称瀑布流。  
  - 图片统一圆角，悬停可 overlay 标题或无。

- **交互**  
  - Hover：图片放大 1.05 + 圆角保持，overflow hidden。  
  - 点击：可弹出 lightbox，打开/关闭带 scale + opacity 动画。

---

### 3.10 页脚

- **布局**  
  - 多列：友情链接、联系方式、备案号、版权等。  
  - 背景深色（如 #1A1A2E），文字浅色，链接 hover 用强调色。

- **动效**  
  - 无需复杂动效，保持稳定可读。

---

## 四、全局交互与动效规范

| 类型 | 建议 |
|------|------|
| **页面加载** | 首屏 Hero 先出现，其余区块按滚动或一次性轻量淡入，避免整页同时闪入。 |
| **滚动** | 导航栏在滚动过 Hero 后由透明变为实色（见 3.1）。可选：部分区块轻微视差。 |
| **进入视口** | 区块采用「淡入 + 短距离上移」或「淡入 + 轻微 scale」，duration 0.5–0.6s，ease-out。 |
| **按钮/链接** | Hover 统一 0.2s ease；可配合下划线展开、颜色变化或背景变化。 |
| **焦点** | 所有可聚焦元素具备清晰 focus 样式（outline 或 box-shadow），符合无障碍。 |

## 五、响应式断点与适配

| 断点 | 宽度 | 说明 |
|------|------|------|
| 手机 | 320–767px | 单列、导航收为汉堡菜单、Banner 高度约 50vh、字体与间距缩小。 |
| 平板 | 768–1023px | 两列网格、导航可仍为汉堡或收起的下拉。 |
| 桌面 | 1024px+ | 完整多列布局、导航全部展开、最大内容宽 1200px 居中。 |

## 六、组件清单（供前端实现）

1. **Header**：Logo、主导航、搜索图标、移动端菜单、滚动态样式切换。  
2. **Hero**：背景图/渐变、标题/副标题/CTA、轮播（可选）、首屏进入动画。  
3. **IconCard**：图标、标题、副标题、hover 上浮与阴影。  
4. **SectionTitle**：区块标题 + 可选副标题，统一字号与间距。  
5. **NewsList**：日期、标题、摘要、hover 高亮、stagger 进入。  
6. **StatBlock**：大数字 + 说明、CountUp 动画、视口触发。  
7. **FeatureCard**：图、标题、描述、链接、hover 图缩放与阴影。  
8. **CTAStrip**：全宽色块、多按钮、hover 反馈。  
9. **ImageGrid**：图片网格、hover 放大、可选 lightbox。  
10. **Footer**：多列链接与文案、深色主题。

## 八、实现建议（给前端）

- **技术选型**：纯 HTML/CSS/JS 即可；若用框架，组件可按第六节拆分。  
- **动画**：CSS `transition`/`transform` 为主，CountUp、视口检测可用少量 JS 或 Intersection Observer。  
- **性能**：图片懒加载、首屏外区块可延迟执行进入动画。  
- **无障碍**：语义化标签、焦点顺序、对比度与 focus 样式按第二节与第四节执行。

如需，可再基于本规范输出「单页 HTML 原型」或「逐区块的样式代码片段」便于直接落地实现。
